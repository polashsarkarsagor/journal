Learning Paradigms

How Machines Learn

Machine Learning is broadly divided into three paradigms, each suited to different types of problems and data.

01

Supervised Learning

The model learns from labeled training data — each input is paired with a correct output. The goal is to learn a mapping function that predicts outputs for unseen inputs.

Think of it like learning with a teacher who tells you if each answer is right or wrong.

Example Email labeled "spam" or "not spam" trains a spam classifier. Given new emails, it predicts the label.
Classification Regression Prediction
02

Unsupervised Learning

The model works with unlabeled data, discovering hidden patterns, structures, and groupings on its own. There's no explicit "correct answer" to guide learning.

Think of it like exploring a new city without a map — you discover patterns from experience.

Example A retailer groups customers by purchasing behavior without predefined categories, revealing natural segments.
Clustering Dimensionality Reduction Anomaly Detection
03

Reinforcement Learning

An agent learns to make decisions by interacting with an environment. It receives rewards for good actions and penalties for bad ones, learning a policy to maximize long-term reward.

Think of it like training a dog — reward good behavior, discourage bad behavior over time.

Example AlphaGo learned to play Go by playing millions of games against itself, improving through win/loss rewards.
Game Playing Robotics Autonomous Systems
Vocabulary

Key Terminology

The language of machine learning — terms you'll encounter in every algorithm and paper.

Training Data

The dataset used to fit the model's parameters. The model learns patterns directly from this data, so its quality and size critically affect performance.

Features

Individual measurable properties of the data used as inputs. For example, in predicting house prices, features might be size, location, and number of bedrooms.

Labels

The correct output values in supervised learning — what the model is trying to predict. Labels are provided in training data but unknown at prediction time.

Overfitting

When a model learns the training data too well, including noise and random fluctuations. It performs excellently on training data but poorly on unseen test data.

Underfitting

When a model is too simple to capture the underlying pattern. It performs poorly on both training and test data — often caused by too few features or low model complexity.

Bias–Variance Tradeoff

Bias = error from overly simple assumptions (underfitting). Variance = error from sensitivity to small fluctuations (overfitting). Good models balance both.

Epoch

One complete pass through the entire training dataset. Models are typically trained for many epochs, gradually improving with each pass.

Loss Function

A mathematical measure of how wrong the model's predictions are. Training aims to minimize the loss. Common examples: Mean Squared Error, Cross-Entropy.

Gradient Descent

An optimization algorithm that adjusts model parameters in the direction that reduces the loss function most steeply, like rolling downhill to find the lowest point.
